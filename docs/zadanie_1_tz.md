# Практическая работа №1
## Техническое задание. Тема 7: Система учёта электроинструмента на предприятии

### Назначение проекта
Веб-приложение предназначено для учёта электроинструмента на предприятии. Система должна хранить список инструментов, показывать их текущее состояние, фиксировать выдачу сотрудникам и регистрировать отправку в ремонт.

### Описание предметной области
На предприятии используется большое количество электроинструмента, который должен храниться по инвентарным номерам. Без цифрового учёта сложно понять, где находится инструмент, кому он выдан и когда был отправлен в ремонт. Проект решает эту задачу в одной системе.

### Пользователи системы
- Кладовщик - добавляет инструмент и контролирует наличие.
- Сотрудник/мастер - получает инструмент в работу.
- Ответственный за ремонт - фиксирует ремонт и обслуживание.

### Функциональные требования
- Добавление инструмента с инвентарным номером.
- Просмотр списка инструментов.
- Фильтрация по статусу и поиск.
- Выдача инструмента сотруднику.
- Возврат инструмента на склад.
- Перевод инструмента в ремонт.
- Просмотр журнала ремонта.
- Получение списка инструментов через JSON API.

### Нефункциональные требования
- Приложение должно запускаться локально без сложной настройки.
- Интерфейс должен быть понятным для пользователя без технической подготовки.
- Данные должны храниться в SQLite-файле.
- Ответ на запросы должен происходить без заметных задержек при учебном объёме данных.

### Технологический стек
| Технология | Назначение |
|---|---|
| Python | Основной язык серверной части. |
| Flask | Обработка HTTP-запросов и маршрутов. |
| SQLite | Хранение данных об инструментах, выдаче и ремонте. |
| HTML | Структура пользовательской страницы. |
| CSS | Оформление интерфейса. |

### Макет интерфейса
В текущей версии интерфейс реализован в виде одной страницы. На ней расположены блок статистики, форма добавления инструмента, фильтр, таблица инструментов и журнал ремонта.

### Предварительное описание API
| Метод и путь | Параметры | Назначение | Ответ |
|---|---|---|---|
| `GET /` | status, search | Открывает главную страницу. | HTML. |
| `POST /tools` | inventory_number, name, category, location, condition | Добавляет инструмент. | Redirect на главную страницу. |
| `POST /tools/<id>/issue` | employee, planned_return_at | Выдаёт инструмент. | Redirect. |
| `POST /tools/<id>/return` | id в URL | Возвращает инструмент. | Redirect. |
| `POST /tools/<id>/repair` | description, cost | Отправляет инструмент в ремонт. | Redirect. |
| `GET /api/tools` | status, search | Отдаёт список инструментов. | JSON. |

### Модель данных
```text
tools (1) -> (many) issue_history
tools (1) -> (many) maintenance
```
