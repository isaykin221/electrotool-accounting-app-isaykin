# Дополнение к практической работе. Задания 2-4

Тема 7: Система учёта электроинструмента на предприятии.

Проект представляет собой учебное веб-приложение для ведения списка электроинструмента, контроля выдачи сотрудникам и фиксации ремонта. В качестве основы использована простая архитектура Flask-приложения по аналогии с примером shopping-list-app.

## Задание 2 - Серверная часть

### Краткое описание

Серверная часть реализована на Python с использованием Flask. Сервер принимает HTTP-запросы от браузера, обращается к SQLite-базе данных через файл `database.py`, формирует HTML-страницу или JSON-ответ и отправляет результат клиенту.

### Структура серверной части

```text
electrotool-accounting-app/
|-- app.py                 - маршруты Flask и обработка HTTP-запросов
|-- database.py            - подключение к SQLite и функции работы с данными
|-- electrotools.db         - файл базы данных SQLite
|-- templates/index.html    - HTML-шаблон страницы
`-- static/styles.css       - стили клиентской части
```

```text
Браузер пользователя -> app.py -> database.py -> electrotools.db
Браузер пользователя <- HTML/JSON <- app.py <- database.py
```

### Основные функции сервера

| Функция/маршрут | Параметры | Что делает | Что возвращает |
|---|---|---|---|
| `index()`, `GET /` | `status`, `search` | Получает список инструментов, статистику и журнал ремонта. | HTML-страницу. |
| `create_tool()`, `POST /tools` | `inventory_number`, `name`, `category`, `location`, `condition` | Добавляет инструмент в таблицу `tools`. | Перенаправление на `/`. |
| `issue(tool_id)`, `POST /tools/<id>/issue` | `tool_id`, `employee`, `planned_return_at` | Выдаёт инструмент сотруднику. | Перенаправление на `/`. |
| `return_item(tool_id)`, `POST /tools/<id>/return` | `tool_id` | Возвращает инструмент на склад. | Перенаправление на `/`. |
| `repair(tool_id)`, `POST /tools/<id>/repair` | `tool_id`, `description`, `cost` | Отправляет инструмент в ремонт. | Перенаправление на `/`. |
| `api_tools()`, `GET /api/tools` | `status`, `search` | Отдаёт список инструментов в JSON. | JSON с количеством и массивом `items`. |

### Примеры запросов и ответов

```http
GET /api/tools?status=in_stock
```

```json
{
  "count": 1,
  "items": [
    {
      "id": 1,
      "inventory_number": "EL-001",
      "name": "Аккумуляторная дрель Bosch",
      "category": "дрель",
      "location": "склад 1",
      "responsible": "",
      "condition": "исправен",
      "status": "in_stock"
    }
  ]
}
```

```http
POST /tools
inventory_number=EL-004&name=Шуруповёрт Интерскол&category=шуруповёрт&location=склад 1&condition=исправен
```

Ответ сервера: `302 Redirect -> /`.

## Задание 3 - Клиентская часть

### Краткое описание

Клиентская часть представлена одной HTML-страницей `templates/index.html` и файлом стилей `static/styles.css`. Страница позволяет пользователю работать с электроинструментом без ручного выполнения SQL-запросов.

### Основные экраны и блоки

| Блок | Назначение |
|---|---|
| Верхний блок со статистикой | Показывает количество инструментов по статусам. |
| Форма добавления | Позволяет внести новый инструмент. |
| Фильтр | Выполняет поиск и фильтрацию по статусу. |
| Таблица инструментов | Показывает данные и кнопки действий. |
| Журнал ремонта | Отображает записи по ремонту. |

### Навигация

```text
Главная страница /
  |-- Добавить инструмент -> POST /tools -> возврат на /
  |-- Применить фильтр -> GET /?status=...&search=...
  |-- Выдать инструмент -> POST /tools/<id>/issue -> возврат на /
  |-- Вернуть инструмент -> POST /tools/<id>/return -> возврат на /
  `-- Отправить в ремонт -> POST /tools/<id>/repair -> возврат на /
```

### Работа кнопок

- `Добавить` создаёт новую запись в базе данных.
- `Показать` применяет поиск и фильтр.
- `Выдать` закрепляет инструмент за сотрудником.
- `Вернуть` меняет статус инструмента на `in_stock`.
- `В ремонт` создаёт запись о ремонте и меняет статус на `repair`.

### Запросы клиента к серверу

| Действие | Запрос |
|---|---|
| Открытие страницы | `GET /` |
| Фильтрация | `GET /?status=issued&search=Makita` |
| Добавление | `POST /tools` |
| Выдача | `POST /tools/1/issue` |
| Возврат | `POST /tools/2/return` |
| Ремонт | `POST /tools/3/repair` |
| Получение JSON | `GET /api/tools` |

### Пользовательский сценарий

1. Пользователь открывает главную страницу приложения.
2. В форме добавления вводит: `EL-004`, `Шуруповёрт Интерскол`, `шуруповёрт`, `склад 1`, `исправен`.
3. Нажимает `Добавить`. Сервер сохраняет запись и возвращает пользователя на главную страницу.
4. Пользователь находит новый инструмент и вводит ФИО сотрудника для выдачи.
5. Нажимает `Выдать`. Статус инструмента становится `issued`.
6. После возврата инструмента пользователь нажимает `Вернуть`. Статус становится `in_stock`.
7. Если инструмент сломался, пользователь вводит причину и нажимает `В ремонт`.

## Задание 4 - Базы данных

### Краткое описание

Для хранения данных используется SQLite. База находится в файле `electrotools.db`. В ней хранятся сведения об инструментах, история выдачи сотрудникам и журнал ремонта.

### Структура базы данных

```text
tools (1) -> (many) issue_history
tools (1) -> (many) maintenance
```

### Таблица tools

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER | Первичный ключ. |
| `inventory_number` | TEXT | Уникальный инвентарный номер. |
| `name` | TEXT | Название инструмента. |
| `category` | TEXT | Категория. |
| `location` | TEXT | Место хранения. |
| `responsible` | TEXT | Ответственный сотрудник. |
| `condition` | TEXT | Состояние инструмента. |
| `status` | TEXT | `in_stock`, `issued` или `repair`. |
| `created_at` | TEXT | Дата создания записи. |

### Таблица issue_history

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER | Первичный ключ. |
| `tool_id` | INTEGER | Внешний ключ на `tools.id`. |
| `employee` | TEXT | Сотрудник, получивший инструмент. |
| `issued_at` | TEXT | Дата выдачи. |
| `planned_return_at` | TEXT | Плановая дата возврата. |
| `returned_at` | TEXT | Фактическая дата возврата. |

### Таблица maintenance

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER | Первичный ключ. |
| `tool_id` | INTEGER | Внешний ключ на `tools.id`. |
| `started_at` | TEXT | Дата начала ремонта. |
| `description` | TEXT | Описание ремонта. |
| `cost` | REAL | Стоимость ремонта. |
| `status` | TEXT | `open` или `closed`. |

### Примеры SQL-запросов

```sql
SELECT * FROM tools ORDER BY created_at DESC;
```

```sql
SELECT * FROM tools WHERE status = 'issued';
```

```sql
INSERT INTO tools (inventory_number, name, category, location, condition)
VALUES ('EL-004', 'Шуруповёрт Интерскол', 'шуруповёрт', 'склад 1', 'исправен');
```

```sql
SELECT tools.inventory_number, tools.name, maintenance.description, maintenance.cost
FROM maintenance
JOIN tools ON tools.id = maintenance.tool_id;
```

```sql
SELECT status, COUNT(*) AS count
FROM tools
GROUP BY status;
```
